## Kontribusi & Commit Guidelines

### Branch Naming Convention
```
feature/nama-fitur          # Untuk fitur baru
bugfix/nama-bug             # Untuk perbaikan bug
docs/update-dokumentasi     # Untuk dokumentasi
```

### Commit Message Format
```
[JENIS] Deskripsi singkat

Deskripsi detail (opsional)

Example:
[FEATURE] Add service management module
[BUGFIX] Fix calculation error in transaction
[DOCS] Update README with installation guide
```

### Code Review Process
1. Push code ke branch fitur
2. Buat Pull Request dengan deskripsi jelas
3. Tunggu review dari minimal 1-2 anggota tim
4. Perbaiki feedback jika ada
5. Merge setelah approval
